# Feature Brief: MinerU + PaddleOCR 文本补全融合

## 1. Overview
当前转换流程依赖 MinerU 结构化结果，但在部分“实际含文字的图片区域”中，MinerU 可能只输出图片块而未提取文字，导致 PPT 结果文本缺失。该功能引入 PaddleOCR 对整页进行文字识别，并与 MinerU 结果做规则融合：若 OCR 文本框与 MinerU 文本框发生重叠（任意像素相交），则保留 MinerU 文本；若无重叠，则补充 OCR 文本，形成最终文本合集用于后续生成。功能以 CLI/GUI 开关方式上线。

## 2. Goals
- 将“MinerU 漏识别文本”的召回率提升 **≥20%**（以标注样本集评估）。
- 控制 OCR 引入的新增误识别率 **≤5%**。
- 在开启该功能时，单页平均处理耗时增幅 **≤25%**。
- 通过开关机制确保用户可按场景启停，不影响默认流程可控性。

## 3. User Stories

- **US-001: 漏识别文本补全**
  - **As a** 使用 MinerU2PPT 的用户, **I want to** 在 MinerU 漏掉文字时自动补齐, **so that I can** 获得更完整的 PPT 文本内容。
  - **Acceptance Criteria:**
    - [ ] 开启功能后，样本集中的漏识别文本被补齐，召回率提升达到目标。
    - [ ] 补齐后的文本参与最终 PPT 生成，而不是仅用于调试输出。

- **US-002: 融合冲突可预期**
  - **As a** 使用者, **I want to** 在 OCR 与 MinerU 冲突时始终优先 MinerU, **so that I can** 保持与既有识别逻辑一致且结果稳定。
  - **Acceptance Criteria:**
    - [ ] 当 OCR 文本框与 MinerU 文本框任意像素相交时，最终结果中该区域使用 MinerU 文本。
    - [ ] 当 OCR 文本框与 MinerU 文本框无重叠时，OCR 文本被纳入最终文本合集。

- **US-003: 功能可控启停**
  - **As a** CLI/GUI 用户, **I want to** 通过参数或界面开关控制 OCR 融合, **so that I can** 根据精度/性能需求选择是否启用。
  - **Acceptance Criteria:**
    - [ ] CLI 提供显式开关参数，默认行为和启用行为可区分。
    - [ ] GUI 提供对应开关项，行为与 CLI 对齐。
    - [ ] 关闭开关时，系统行为与当前版本一致（不引入 OCR 融合结果）。

## 4. Functional Requirements
- **FR-1（US-001）**：系统在开启功能时，必须对输入页面执行 PaddleOCR 文本检测与识别，生成 OCR 文本框及文本内容集合。
- **FR-2（US-002）**：系统必须从 MinerU 输出中提取文本块集合，作为主文本基准集合。
- **FR-3（US-002）**：系统必须执行融合规则：
  1) OCR 文本框与任一 MinerU 文本框发生任意像素相交 -> 丢弃该 OCR 文本；
  2) OCR 文本框与所有 MinerU 文本框均不相交 -> 保留该 OCR 文本。
- **FR-4（US-001/US-002）**：系统必须输出“MinerU 文本 + 可保留 OCR 文本”的最终文本合集，并用于后续渲染生成流程。
- **FR-5（US-003）**：系统必须提供 CLI 开关与 GUI 开关，以启用/禁用 OCR 融合功能。
- **FR-6（US-003）**：系统在功能关闭时，必须保持现有 MinerU-only 处理路径，输出与当前流程一致。
- **FR-7（US-001）**：系统应记录可用于评估的统计信息（如 OCR 候选数、被重叠过滤数、最终补充数），用于验收与回归比较。
- **FR-8（US-001/US-003）**：当 OCR 执行失败时，系统必须回退至 MinerU-only 路径并给出可诊断错误信息，不中断整批任务流程。

## 5. Non-Goals (Out of Scope)
- 不在本阶段实现基于语义相似度/语言模型的高级文本冲突消解。
- 不在本阶段对 OCR 结果做复杂版面重排（例如跨栏重组、阅读顺序重建）。
- 不在本阶段替换 MinerU 主识别能力，仅作为补全增强。
- 不在本阶段承诺对所有语言做同等质量优化（以当前配置支持语言为准）。

## 6. Success Metrics
- 漏识别文本召回率提升 **≥20%**。
- OCR 引入的新增误识别率 **≤5%**。
- 单页平均处理耗时增幅 **≤25%**。
- 在关闭开关时，与现有版本输出一致性 **100%**（回归样本集）。

## 7. Open Questions
1. OCR 支持语言范围是否固定（如中英）还是需要可配置？
2. “重叠判定”是否只基于矩形框相交，还是后续需扩展为最小面积阈值（当前按任意像素相交）？
3. 对 OCR 低置信度文本是否需要增加阈值过滤（当前规格未强制）？
4. 评估样本集规模与标注口径如何定义（用于召回率/误识别率统一计算）？
5. GUI/CLI 开关的默认值是否都设为关闭（当前倾向可控上线）？
